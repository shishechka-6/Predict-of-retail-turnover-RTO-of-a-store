# Retail Turnover (RTO) Forecasting for Pyaterochka Stores

Система предназначена для месячного прогнозирования розничного товарооборота (РТО) 20 615 магазинов сети «Пятёрочка» по истории за 10 предыдущих месяцев. 

Основная цель — предсказание выручки каждого магазина на ноябрь — месяц, отсутствующий в обучающей выборке, что моделирует реальную операционную задачу финансового и операционного департаментов сети.

В основе решения лежит ансамбль из трёх градиентных бустингов (LightGBM × 2 + CatBoost), обученных под MAPE-loss. Архитектура адаптирована под задачу:

- обучение проводится на ~144 K синтетических пар вида «(магазин, target_month t) → РТО месяца t», где признаки построены только из месяцев `1..t−1` — это учит модель универсальному one-step-ahead прогнозу, применимому к ноябрю;

- ключевой приём: `sample_weight = 1/y_true` при MSE-loss → оптимизация эквивалентна MAPE без кастомных objective-функций;

- ансамблирование реализовано через OOF-стекинг по 4 фолдам (`t ∈ {7, 8, 9, 10}`) с multi-start Nelder-Mead подбором неотрицательных весов в лог-пространстве;

- применяется per-store клиппинг `[lo·lag_1, hi·lag_1]` с подбором границ на OOF — снижает «выбросы» прогноза на нестабильных магазинах.

Модель достигает **MAPE = 5.12 %** на валидации (94.88 баллов из 100 по формуле `score = 100 − MAPE`), что в 9 раз ниже ошибки бейзлайна (45.66 %) и более чем на 40 баллов превышает порог успешного решения.

## Запуск (Inference + Training)

1. Откройте основной ноутбук:

```bash
jupyter notebook solution_best.ipynb
```

2. Последовательно выполните все ячейки:
   - загрузка `train.csv` и `test.csv`;
   - feature engineering — 50 признаков на магазин (лаги, rolling-статистики, тренды, региональные агрегаты, Fourier-кодирование);
   - OOF-стекинг по 4 фолдам с 3 моделями × 3 сидами;
   - подбор весов ансамбля и клиппинга на OOF;
   - финальное обучение на всех `target_month ∈ {4..10}` и прогноз ноября;
   - запись `test.csv` с предсказаниями.

```text
test.csv
```

Время полного прогона: ~20 минут на MacBook Pro M1 (используются все ядра CPU). На сервере с GPU/большим CPU быстрее.

## Особенности решения

- **Конструкция обучающей выборки.** На каждое сочетание `(магазин, target_month t)` для `t ∈ {4..10}` строится одна строка с признаками только из месяцев `< t`. Итого ~144 K обучающих примеров. Модель учится универсальному one-step-ahead прогнозу, применимому к ноябрю (`t = 11`) без переобучения.

- **Sample weights = 1/y.** Главный рычаг для MAPE-loss. Простое перевзвешивание выборки делает обычный MSE эквивалентом MAPE без необходимости кастомных objective-функций или log-преобразований.

- **OOF-стекинг вместо подбора весов на одном hold-out.** Прогнозы каждой из 3 моделей собираются по 4 фолдам `t ∈ {7, 8, 9, 10}` (обучение на `< t`, прогноз на `t`), затем веса стекера подбираются Nelder-Mead с 4 разными стартовыми точками — это избегает локального оптимума, в который single-fold оптимизация ранее застряла.

- **Per-store clipping.** Прогноз клиппируется в окно `[lo·lag_1, hi·lag_1]`, где `lo, hi` подбираются на OOF по сетке. Это блокирует «выбросы» прогноза в условиях, когда модель экстраполирует слишком оптимистично/пессимистично.

- **No-leak финальное обучение.** Финальные модели переобучаются на полной выборке с фиксированным числом итераций (медиана `best_iteration` из OOF), без `eval_set` и без early stopping — это исключает leakage от валидационного фолда.
